# SPX 1DTE 裸賣 Put 機器人

每個交易日賣出一檔 **隔日到期、約 7 delta 的 SPX put**，收權利金，
同時預先掛好一張 **GTC 自動停利單**（權利金跌到剩 22% 就自動買回，鎖住 78% 獲利）。
下完單程式就結束 —— 剩下的交給 IBKR 幫你顧。

```
連上 IBKR → 確認開盤/時段/帳戶 → 找 7 delta 的 put → 賣出 + 掛好停利 → 收工
```

---

# 🟢 完全沒經驗？照這份走（Windows 版）

> 這一段是給「沒寫過程式、沒用過終端機」的人。**每一步照做就好，不用懂原理。**
> 全程大概 20–30 分鐘。用到的東西都是免費的，而且都在**模擬倉**（假錢）跑，不會動到真錢。

## 你會裝兩個東西

1. **Node.js** —— 跑這支程式的引擎（像是「安裝 Word 才能開 .docx」的概念）。
2. **IB Gateway** —— IBKR 官方的連線程式，程式透過它去下單。

裝好之後，打三行指令就能跑起來。

---

## 步驟 1：裝 Node.js

1. 開瀏覽器到 👉 <https://nodejs.org>
2. 點左邊那顆綠色大按鈕（寫著 **LTS**，例如「20.x.x LTS」）。它會下載一個 `.msi` 檔。
3. 打開下載的 `.msi`，一路按 **Next**（保持預設就好，不用改任何選項），最後按 **Install**。
   - 中間若跳出「是否允許這個 App 變更你的裝置？」按 **是**。
4. 裝完按 **Finish**。

✅ **怎麼確認裝好了？** 看步驟 2。

---

## 步驟 2：打開「終端機」，確認 Node 裝好

「終端機」就是一個打指令的黑框框。Windows 打開方法：

1. 按鍵盤左下角的 **⊞ Windows 鍵**。
2. 直接打字 `powershell`。
3. 點出現的 **Windows PowerShell**。

會跳出一個藍色（或黑色）視窗。在裡面打這行，然後按 **Enter**：

```powershell
node -v
```

- 如果印出像 `v20.11.1` 之類的版本號 → 🎉 成功，繼續下一步。
- 如果說「無法辨識 node」→ 把電腦**重新開機**再試一次（安裝後要重開才會生效）。

---

## 步驟 3：裝 IB Gateway，登入「模擬倉」

1. 下載 IB Gateway 👉 <https://www.interactivebrokers.com/en/trading/ibgateway-stable.php>
   （選 Windows 版，下載後一路 Next 裝好，跟裝 Node 一樣。）
2. 打開 IB Gateway。第一個畫面會問你要登入哪種帳戶：
   - 選 **Paper Trading（模擬交易 / 模擬倉）** ← 一定要選這個！
   - 輸入你的 IBKR 帳號密碼登入。
     （還沒有 IBKR 帳號的話，用這條連結免費開一個 👉 <https://ibkr.com/referral/shanti102> —— 開戶時會附一個 paper 帳號。）
3. 登入後，會看到一個小小的視窗。點上方選單 **Configure ⚙️ → Settings**。
4. 左邊清單點 **API → Settings**，右邊確認這幾項：
   - ✅ 勾 **Enable ActiveX and Socket Clients**
   - **Socket port** 要是 **4002**（這是模擬倉的預設埠，通常本來就是）
   - ✅ 勾 **Allow connections from localhost only**
   - 把 **Read-Only API** 的勾**拿掉**（打勾的話程式只能看、不能下單）
5. 按 **OK / Apply** 存檔。

⚠️ **IB Gateway 要一直開著**，程式才連得上。你可以把它縮到最小，但別關掉。

> 💡 模擬倉是假錢，儘管放心試。玩到滿意再考慮實倉（見最後一節）。

## 步驟 3.5：IB 行情訂閱（可跳過，程式會自動處理）

IBKR 的即時報價**要另外付費訂閱**。沒訂閱的話拿到的是**延遲約 15 分鐘**的報價。

**這支程式會自動判斷，你什麼都不用設**：連線時先試即時行情，如果你的帳戶沒訂閱，
它會印一行「⚠️ 偵測到沒有即時行情訂閱，改用延遲報價，繼續執行」然後照常跑完 ——
所以**你不訂閱也能連線、跑完整流程、下單**。拿來在模擬倉「看它會動」完全夠用。

**如果你要玩真的（實倉、或想要準確的即時價），可以去買這兩項行情：**

1. 打開 IBKR 的 **Client Portal**（網頁登入 <https://www.interactivebrokers.com>）。
2. 右上 **帳號圖示 → Settings（設定）→ Market Data Subscriptions（行情訂閱）**。
3. 訂這兩個（個人非專業戶，一個月大約幾塊美金）：
   - **OPRA (US Options Exchanges)** —— 美股**選擇權**即時報價（挑 delta、算權利金要用）。
   - **Cboe One Add-On**（或含 **S&P 指數** 的指數行情）—— **SPX 指數**即時價。
4. 訂閱綁在你的**實倉**帳號上。要讓**模擬倉**也吃得到，另外開一個開關：
   **Settings → 找到 "Share market data with paper account"（與模擬帳戶共享行情）→ 打開。**

訂好之後**什麼都不用改** —— 程式下次連線會自動抓到即時行情，改印一行「📡 使用即時行情」。

## 步驟 6：跑起來！

確認 IB Gateway 還開著、已登入模擬倉，然後打：

```powershell

```

程式會連上 Gateway、印出你的帳戶淨值，然後依照當下情況動作。

**如果現在是美股開盤時段**，你會看到它挑一檔 put、賣出、掛好停利，長這樣：

## 🧪 想先「試跑」但不真的下單？

打開 `spx-1dte-naked-put.js`（用記事本或 VS Code 都行），
找到最上面 `CONFIG` 裡的這行，把 `false` 改成 `true`：

```js
  dryRun: true,   // true = 只印出「會下什麼單」但不真的送出
```

存檔後 `npm start`，它會走完整個流程、印出「會下的單」，但**不會真的送出去**。
確認一切正常後再改回 `false`。

# ⚙️ 參數怎麼調（都在 `spx-1dte-naked-put.js` 最上面的 CONFIG）

| 參數 | 預設 | 意思 |
|---|---|---|
| `port` | `4002` | **4002=模擬倉**、`4001`=實倉、`7497`=TWS 模擬 |
| `marketDataType` | `'auto'` | 自動:連上先試即時、沒訂閱就退延遲(通常不用動)。要強制可填 `1`=即時 / `3`=延遲 |
| `targetDelta` | `0.07` | 要賣幾 delta 的 put（越小越價外、越安全、收越少） |
| `profitTarget` | `0.78` | 停利幅度（0.78 = 賺到 78% 就自動買回） |
| `quantity` | `1` | 每次賣幾口 |
| `minPremium` | `1.0` | 權利金太低就不做（$/股） |
| `dryRun` | `false` | `true` = 只印不下單（試跑用） |

# 🆘 遇到問題？（常見狀況）

| 畫面訊息 | 意思 / 怎麼辦 |
|---|---|
| `無法辨識 'node'` / `'npm'` | Node 沒裝好，或裝完沒重開機。重開機再試；還不行就重裝步驟 1。 |
| `連線逾時` / `connect ECONNREFUSED` | IB Gateway **沒開**、**沒登入**，或 **API 沒勾**。回步驟 3 檢查，確認 Gateway 開著且登入模擬倉。 |
| `⚠️ IBKR [2104/2106/2158]` 之類 | 這是「資料連線正常」的**通知**，不是錯誤，可以忽略。 |
| 印了帳戶就停，說「休市/非進場時段」 | 正常！代表連線 OK，只是還沒到下單時間。等美股開盤時段再跑。 |
| `取得 SPX 現價逾時` / 找不到合約 | 休市時常見。或 `marketDataType` 設成 `1` 但**沒買行情訂閱** → 改回 `3`（延遲，免費）。 |
| 帳戶淨值是 0 或怪怪的 | 確認你登入的是**模擬倉**、且 Gateway 已完全登入好（不是還在載入）。 |

# ⏰ 每天自動跑（進階，選用）

這支程式是「跑一次就結束」，所以請**系統排程器**每個交易日的進場時段叫它一次即可。

**Windows —— 用「工作排程器」：**
1. Windows 鍵打 `工作排程器`（Task Scheduler）打開它。
2. 右邊 **建立基本工作** → 取名 → 觸發程序選 **每天** → 設一個進場時段內的時間（例如台灣時間 22:00）。
3. 動作選 **啟動程式**：
   - 程式或指令碼：`npm`（或 `C:\Program Files\nodejs\npm.cmd`）
   - 引數：`start`
   - 開始位置：你的資料夾路徑（例如 `C:\Users\你\Desktop\spx-bot`）
4. 完成。記得 **IB Gateway 要保持開著**它才連得上。

（Mac / Linux 用 crontab，例如週一到五台灣時間 22:00：
`0 22 * * 1-5 cd /路徑/spx-bot && npm start >> run.log 2>&1`）

# ⚠️ 重要提醒

- **先在模擬倉（4002）跑到滿意再說。** 要換實倉才把 `CONFIG.port` 改成 `4001`，並**自負盈虧**。
- 賣 put 的本質是「用保證金去收隔夜的時間價值」，大跌時會有帳面虧損 —— 這是策略的常態，不是程式壞掉。
- 掛出去的停利單是 **GTC（掛到成交或手動取消為止）**，程式關掉不影響它。
- 要手動停掉/查看部位，直接到 IB Gateway 或 TWS 操作。
- 這不是投資建議。你要為自己帳戶的每一筆單負責。
